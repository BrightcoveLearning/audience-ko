페이지 내용

    단일 사인온과 대상 통합

    이 항목에서는 Single Sign-On과 함께 사용할 대상 모듈을 구성하는 방법에 대해 설명합니다.

    Audience 모듈은 사용자가 SSO (Single Sign-On) 를 사용하여 시스템에 로그인한 경우 보기 활동을 추적할 수 있는 기능을 제공합니다. 이 시나리오에서는 데이터 보기 마케팅 자동화 플랫폼에 전송 되지 않습니다. 대상 SSO 구성을 사용하면 SAML 특성이 오디언스로 전달되어 뷰어를 식별합니다. 이에 대한 일반적인 사용 사례 중 하나는 내부 교육 비디오입니다. 직원은 비디오를 시청하기 위해 로그인해야 할 수 있으며 시청 활동을 추적한 다음 Audience 모듈 내에서 볼 수 있습니다. 오디언스를 SSO와 함께 사용하려면 다음 단계가 필요합니다.

    1. 단일 사인온 소프트웨어 구성
    2. SSO 특성에 액세스하도록 응용 프로그램 구성
    3. 대상 SSO 연결 설정
    4. 통합 구성
    5. 오디언스가 활성화된 플레이어 만들기
    6. 비디오 데이터 보기

    SSO 소프트웨어 구성

    SSO 소프트웨어를 설치 및 구성하는 단계는 이 항목의 범위를 벗어납니다. SSO 소프트웨어는 사용자가 Video Cloud 비디오를 보기 전에 자신을 인증해야 하도록 구성해야 합니다. SSO 소프트웨어는 외부 응용 프로그램에서 액세스할 수 있도록 일부 사용자 속성을 노출하도록 구성해야 합니다. 예를 들어, 이 Okta SSO 구성은 세 가지 특성FirstName , LastName및 을 표시합니다Email .

    이 항목에서Email속성은 Audience 모듈의 뷰어를 식별하는 데 사용됩니다.

    참고: SSO 소프트웨어가 여러 특성을 노출하도록 구성할 수 있지만 Audience 는 단일 속성만 사용하여 뷰어를 추적합니다.

    SSO 특성에 액세스하도록 응용 프로그램 구성

    Video Cloud 비디오를 보는 데 사용할 애플리케이션 또는 웹 사이트는 SSO 속성을 JavaScript를 사용하여 오디언스가 사용할 수 있도록 구성해야 합니다. 이 구성은 사용 중인 응용 프로그램 또는 웹 사이트에 따라 다릅니다. 다음 예에서는 갤러리에 SSO 액세스 제어 프로필이 만들어졌습니다. 구성의 일부로Email SSO 특성이 노출되고 갤러리 사이트 내에서 사용할 수 있습니다. Audience 모듈에 대한 시청자를 식별하는 데 사용되는 값입니다.

    참고: 갤러리에서 액세스 제어 프로파일 구성에 대한 자세한 내용은액세스 제어 프로파일 구성을참조하십시오.
    참고: 여러 속성을 노출하도록 갤러리를 구성할 수 있지만 Audience 는 단일 속성만 사용하여 뷰어를 추적합니다.

    연결 설정

    다음 단계에 따라 Video Cloud와 SSO 공급자 간의 연결을 설정합니다.

    비디오 클라우드와 SSO 간의 연결 설정

    1. 대상 모듈을 열고 왼쪽 탐색에서데이터 연결을 클릭합니다.
    2. 새 연결 추가를클릭합니다.
    3. 통합으로SSO를선택합니다.
      참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
    4. (선택 사항) 연결 이름을입력합니다.
    5. 연결을클릭합니다.

    이제 SSO 연결이 설정되고 사용할 준비가 되었습니다.

    통합 구성

    SSO 연결이 설정되면 통합을 구성할 수 있습니다.

    1. 클릭empty링크 SSO 속성부분. 페이지에서 찾을 JavaScript 값의 이름을 입력합니다. 이 예에서는 SSO 특성을 노출하도록 Gallery가Email구성되었으므로 SSO 특성이 여기에 사용됩니다.

      이 항목에서는 대상 SSO 통합이 보기 활동을 추적하는 데 사용됩니다. 저장을 클릭합니다.

      참고: SSO 속성을비워 둔 다음추가 옵션중 하나를 사용하여 뷰어를 식별할 수도 있습니다.
    2. 기본적으로통합 상태는사용으로설정됩니다. 통합 상태에따라 Video Cloud 비디오 참여 데이터가 저장되는지 여부가 결정됩니다.
      • 사용 - 데이터가 수집됩니다.
      • 사용 안 함 - 데이터가 수집되지 않습니다 (통합이 비활성화됨).
    3. 데이터 보존 정책을 선택합니다. 데이터 보존 정책에 따라 시청 데이터와 잠재 고객 양식 데이터 (Audience 리드 양식을 사용하는 경우) 가 Video Cloud 계정에 보존되는 기간이 결정됩니다. 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다.

    통합을 제거하려면연결 끊기를클릭합니다. 페이지 하단에는 모든 오디언스가 활성화된 플레이어, 연결된 연결 및 관련 리드 양식이 나열됩니다.

    Audience 모듈은 마케팅 플랫폼에 대한 여러 연결을 지원합니다. 추가 통합을 구성하려면 새 연결 추가를클릭합니다.

    시청자 식별을 위한 추가 옵션

    통합이 구성될 때SSO 등록 정보값을 지정하지 않으면 뷰어를 식별할 책임이 있습니다. Audience 모듈에는 다음 정보가 필요합니다.

    설명 URL 매개 변수 매개변수 포함 값
    GUID 란 무엇입니까 (보기 이벤트 추적에 필요함) bcssoguid data-bc-sso-guid string

    이 파라미터는 URL 파라미터 또는 플레이어 임베드 코드를 통해 전달될 수 있습니다. 뷰어 추적이 발생하려면 비디오를 표시하는 페이지에서 이 매개 변수를 URL 매개 변수로 전달해야 합니다.

    참고: 플레이어/플러그인이 URL 매개 변수를 읽으려면 고급 (페이지 내) 포함 코드를 사용해야합니다. 플러그인이 iframe 내부에서 실행되므로 표준 (iframe) embed 코드는 작동하지 않으므로 매개 변수가있는 상위 페이지 URL에 액세스 할 수 없습니다.

    페이지에서 사용할 수 있는 JavaScript 값을 사용하여 대상 플레이어에게 GUID를 전달할 수도 있습니다. 플레이어 설정을 사용하면 잠재고객에게 JavaScript 변수의 이름이 무엇인지 알릴 수 있으며 Audience 는 사용자 지정 GUID로 사용하여 해당 변수를 읽으려고 시도합니다. 이렇게 하려면 오디언스 플레이어를 편집하고 오디언스 플러그인에 대한 플러그인 옵션을 구성합니다.

    위의 예에서, 플레이어 설정이라는samlAttributes페이지에 자바 스크립트 객체가 있다는 것을 오디언스에게 말하고 그것이라는 속성이nameID있습니다. 대상은이를 읽고 사용자 지정 GUID에 대한 값을 사용하려고 시도합니다.

    오디언스가 활성화된 플레이어 만들기

    비디오 참여 데이터는 시청자 지원 비디오 플레이어를 사용하는 경우에만 추적됩니다. 오디언스가 활성화된 브라이트코브 플레이어 생성에 대한 자세한 내용은오디언스가 활성화된 플레이어 만들기를참조하십시오.

    비디오 데이터 보기

    SSO 통합을 사용할 때 비디오 데이터를 보는 데 사용할 수 있는 몇 가지 옵션이 있습니다.

    • 시청자 활동 (이벤트) - 대상 모듈의 일부로 제공되는 이 옵션을 사용하여 Audience 모듈 내의 비디오 데이터를 볼 수 있습니다. 자세한 내용은대상 모듈에서 플레이어 이벤트 보기를참조하십시오.
    • 시청자 프로필 - Audience 모듈의 일부로 제공되는 이 옵션을 사용하여 Audience 모듈 내의 사용자 프로필 값을 기반으로 비디오 데이터를 볼 수 있습니다. 자세한 내용은뷰어 프로파일 정보 보기를참조하십시오.
    • 데이터 내보내기 - Audience 모듈의 일부로 제공되는 이 옵션을 사용하여 비디오 보기 및 리드 데이터를 내보낼 수 있습니다. 자세한 내용은대상 모듈에서 사용자 데이터 내보내기 를참고하십시오.
    • 오디언스 API - 오디언스 API는 프로그래밍 방식으로 동영상 보기 및 리드 데이터를 검색하는 데 사용할 수 있습니다. 자세한 내용은개요: 오디언스 API
