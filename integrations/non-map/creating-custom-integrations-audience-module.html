페이지 내용

    대상 모듈로 사용자 지정 통합 만들기

    이 항목에서는 사용자 지정 통합을 위해 Audience 모듈을 구성하는 방법에 대해 설명합니다.

    Audience 모듈을 통해 Video Cloud는 오디언스 지원 브라이트코브 플레이어의 시청 데이터를 인기 있는 마케팅 자동화 플랫폼으로 동기화할 수 있습니다. Audience 모듈은 사용자 정의 통합을 사용하여 구성할 수도 있습니다. 사용자 지정 통합을 사용하면 보기 데이터가 Video Cloud에 로컬로 저장되고 Audience API는 프로그래밍 방식으로 데이터를 추출하는 데 사용됩니다.

    Audience 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

    • 비디오 조회수 데이터 추적을 지원하는 브라이트코브 플레이어 플러그인
    • Audience API를 사용하여 보기 데이터를 검색하는 사용자 지정 개발

    요구 사항

    대상 모듈을 사용하려면 다음을 수행해야 합니다.

    • 프로페셔널 또는 엔터프라이즈 비디오 클라우드 계정
    • Audience 모듈을 열 때 나타나는 초기 시작 화면에서 필요한 정보를 제출하여 계정에 대한 Audience 모듈을 활성화하십시오.
    • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
    • 리드를 수집하려면리드 양식을 구성하고 플레이어에게할당해야 합니다.
    • 플레이어를 게시 할 때 고급 (페이지 내) embed 코드를 사용하십시오. 플러그인이 iframe 내부에서 실행되므로 표준 (iframe) embed 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수 없습니다.

    연결 설정

    다음 단계에 따라 사용자 지정 통합을 구성합니다.

    1. 대상 모듈을 열고 왼쪽 탐색에서데이터 연결을 클릭합니다.
    2. 새 연결 추가를클릭합니다.
    3. 통합으로 [ 사용자 정의 ] 를 선택합니다.
      참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
    4. (선택 사항) 연결 이름을입력합니다.
    5. 연결을클릭합니다.

    이제 사용자 지정 연결이 설정되고 사용할 준비가 되었습니다.

    통합 구성

    연결이 완료되면 통합을 구성할 수 있습니다.

    1. 기본적으로연결 상태는사용 가능으로 설정됩니다 . 연결 상태에따라 Video Cloud 비디오 참여 데이터가 저장되는지 여부가 결정됩니다.
      • 사용 - 데이터가 수집되고 저장됩니다.
      • 사용 안 함 - 데이터가 수집되지 않습니다 (통합이 비활성화됨).
    2. 데이터 보존 정책을 선택합니다. 데이터 보존 정책에 따라 시청 데이터와 잠재 고객 양식 데이터 (Audience 리드 양식을 사용하는 경우) 가 Video Cloud 계정에 보존되는 기간이 결정됩니다.
      참고: 보존 안 함을 선택하면 데이터가 저장되지 않습니다. API를 사용하여 데이터를 다운로드하면 아무 것도 없습니다.

    오디언스가 활성화된 플레이어 만들기

    비디오 참여 데이터는 시청자 지원 비디오 플레이어를 사용하는 경우에만 저장됩니다. 오디언스가 활성화된 브라이트코브 플레이어 생성에 대한 자세한 내용은오디언스가 활성화된 플레이어 만들기를참조하십시오.

    추적 ID 전달

    Audience 모듈은 추적 ID를 전달하는 기능을 지원합니다. 추적 ID는 캠페인 ID를 추적하는 데 사용할 수 있습니다. 추적 ID는 오디언스가 활성화된 플레이어를 표시하는 URL에 추가하거나 플레이어 임베드 코드에 추가할 수 있습니다.

    • 예: URL에 추가됨
      http://www.myurl.com/video.html?bctrackingid=12345
    • 예: 플레이어 임베드 코드에 추가됨
    <비디오 데이터 - 비디오 ID="6033402539001"
        데이터 계정 = “1486906377”
        데이터 플레이어 = “기본값”
        데이터 포함 = “기본값”
        데이터 애플리케이션 ID
        클래스="비디오-js”
        데이터 BC 추적 ID="12345"
        > <컨트롤/비디오 >
<스크립트 스크립트 스크립트: “> </” >

    시청자 식별

    사용자 지정 통합을 사용하는 경우 뷰어를 식별할 책임이 있습니다. Audience 플러그인에는 다음과 같은 두 가지 주요 정보가 필요합니다.

    설명 URL 매개 변수 매개변수 포함 값
    뷰어가 알려져 있는지 (리드 양식에 필요함) bcknownuser data-bc-known-user 0또는1
    GUID 란 무엇입니까 (보기 이벤트 추적에 필요함) bccustomguid data-bc-custom-guid string
     

    이 값은 URL 매개 변수 또는 플레이어 embed 코드를 통해 전달될 수 있습니다. 뷰어 추적이 이루어지려면 사용자 ID가 비디오를 표시하는 페이지에서 URL 매개 변수로 전달되어야 합니다. 리드 양식에는 알려진 사용자 매개 변수가 필요합니다 (알 수 없는 뷰어에 대해 리드 양식을 조건부로 표시하려는 경우).

    참고: 플레이어/플러그인이 URL 매개 변수를 읽으려면 고급 (페이지 내) 포함 코드를 사용해야합니다. 플러그인이 iframe 내부에서 실행되므로 표준 (iframe) embed 코드는 작동하지 않으므로 매개 변수가있는 상위 페이지 URL에 액세스 할 수 없습니다.

    비디오 데이터 보기

    사용자 지정 통합을 사용할 때 비디오 데이터를 보는 데 사용할 수 있는 몇 가지 옵션이 있습니다.

    • 시청자 활동 (이벤트) - 대상 모듈의 일부로 제공되는 이 옵션을 사용하여 Audience 모듈 내의 비디오 데이터를 볼 수 있습니다. 자세한 내용은대상 모듈에서 플레이어 이벤트 보기를참조하십시오.
    • 데이터 내보내기 - Audience 모듈의 일부로 제공되는 이 옵션을 사용하여 비디오 보기 및 리드 데이터를 내보낼 수 있습니다. 자세한 내용은대상 모듈에서 사용자 데이터 내보내기 를참고하십시오.
    • 오디언스 API - 오디언스 API는 프로그래밍 방식으로 동영상 보기 및 리드 데이터를 검색하는 데 사용할 수 있습니다. 자세한 내용은개요: 오디언스 API
