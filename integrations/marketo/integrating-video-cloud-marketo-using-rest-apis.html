페이지 내용

    REST API를 사용하여 비디오 클라우드와 Marketo의 통합

    이 항목에서는 REST API를 사용하여 Marketo에 사용할 대상 모듈을 구성하는 방법에 대해 설명합니다.

    Audience 모듈을 사용하면 비디오 클라우드가 브라이트코브 플레이어의 시청 데이터를 Marketo로 동기화할 수 있습니다. Marketo에 있는 경우 이 데이터는 세분화, 캠페인 로직, 보고서, 잠재 고객 점수 계산 및 커뮤니케이션 개인 설정에 사용할 수 있습니다.

    참고: Munchkin ID를 사용하여 Marketo와 함께 사용하도록 대상 모듈을 구성하는 방법에 대한 자세한 내용은 Munchkin을사용하여 Marketo와 비디오 클라우드 통합을참조하십시오.

    Marketo와의 Audience 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

    • 비디오 조회수 데이터 추적을 지원하는 브라이트코브 플레이어 플러그인
    • 비디오 클라우드에서 Marketo로 추적 데이터를 전송하는 동기화 프로세스

    요구 사항

    Marketo REST 통합과 함께 대상 모듈을 사용하려면 다음을 수행해야 합니다.

    • Marketo에서 모든 API 권한에 액세스할 수 있는 역할 만들기
    • 런치포인트 서비스 생성
    • 프로페셔널 또는 엔터프라이즈 비디오 클라우드 계정
    • Audience 모듈을 열 때 나타나는 초기 시작 화면에서 필요한 정보를 제출하여 계정에 대한 Audience 모듈을 활성화하십시오.
    • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
    • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내부에서 실행되므로 표준 임베드 (iframe) 게시 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수 없으며 올바른 쿠키를 얻지 못할 수 있습니다.

    API 전용 사용자 및 런치포인트 서비스 생성

    Marketo REST API를 호출하려면 먼저 모든 API에 액세스할 수 있는 새 API 역할을 생성하고 API 전용 사용자를 생성한 다음 클라이언트 애플리케이션을 고유하게 식별하는 서비스를 생성해야 합니다. 이러한 작업을 완료하는 단계는Marketo 설명서에요약되어 있습니다.

    연결 설정

    Marketo 측의 구성 단계가 완료되면 다음 단계에 따라 Video Cloud와 Marketo 간의 연결을 설정하십시오.

    비디오 클라우드와 Marketo 간의 연결 구축

    1. 대상 모듈을 열고 왼쪽 탐색에서데이터 연결을 클릭합니다.
    2. 새 연결 추가를클릭합니다.
    3. Marketo를통합으로 선택합니다.
      참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
    4. (선택 사항) 연결 이름을입력합니다.
    5. 연결을클릭합니다.
    6. 표준 (REST API) 을 선택한 다음저장을클릭합니다.
      참고: Munchkin 통합 사용에 대한 자세한 내용은Munchkin을 사용하여 Marketo와 비디오 클라우드통합을 참조하십시오.

      Marketo 계정에서 여러 값을 붙여넣어야 합니다. 다음 단계를 완료하려면 Marketo 관리자로 로그인해야 합니다. 다음 단계에서는 빈 링크를 클릭하고 적절한 값을 제공합니다.

    7. Marketo 계정 ID - 이 값을 가져오려면 Marketo에서관리자 > 먼치킨을클릭합니다.
    8. Marketo REST API 끝점 URL - 이 값을 가져오려면 Marketo에서관리 > 웹 서비스를클릭합니다.
    9. Marketo REST API ID URL - 이 값을 가져오려면 Marketo에서관리 > 웹 서비스를클릭합니다.
    10. Marketo 런치포인트 클라이언트 ID - 이 값을 가져오려면 Marketo에서관리자 > 런치포인트를클릭한 다음 이전에 생성한 런치포인트에 대한세부 정보 보기링크를 클릭합니다.이 설정의 일부로.
    11. Marketo 런치포인트 클라이언트 비밀 - 이 값을 가져오려면 Marketo에서관리자 > 런치포인트를클릭한 다음 생성한 런치포인트에 대한세부 정보 보기링크를 클릭합니다.이 설정의 일부로 이전.
    12. 설정을 클릭합니다.

    이제 Marketo에 대한 비디오 클라우드 연결이 설정되어 사용할 준비가 되었습니다.

    참고: Marketo 사용자에게 올바른 사용 권한이 할당되지 않으면 Studio에 오류가 표시됩니다. 권한을 확인한 다음 다시 시도하십시오.

    통합 구성

    비디오 클라우드와 Marketo 연결이 완료되면 통합을 구성할 수 있습니다.

    1. 기본적으로연결 상태는사용 가능으로 설정됩니다 . 연결 상태에따라 Video Cloud 비디오 참여 데이터가 Marketo로 전송되는지 여부가 결정됩니다.
      • 사용 - 데이터가 수집되어 Marketo로 전송됩니다.
      • 사용 안 함 - 데이터가 수집되지 않습니다 (통합이 비활성화됨).
    2. 기본적으로알 수 없는 뷰어에 대한 데이터 추적은사용 안 함으로설정됩니다. 이 옵션을 사용하면 알 수 없는 사용자에 대한 데이터가 추적됩니다. 선택한 데이터 보존 정책에 따라 데이터가 보존됩니다. 데이터 보존이보존 안됨으로 설정된 경우 이 필드는 표시되지 않습니다.
    3. 데이터 보존 정책을 선택합니다. 데이터 보존 정책에 따라 캡처한 잠재 고객 정보 데이터가 Marketo뿐만 아니라 Video Cloud 계정에 보존되는지 여부와 기간이 결정됩니다. Video Cloud에 데이터를 보관하면 중단된 동기화로부터 복구할 수 있습니다. 그러나개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다. Video Cloud에 사용자 데이터를 보관할 필요는 없습니다. 데이터는 데이터 보존 기간이 지나면 삭제됩니다.

    Marketo와의 통합을 제거하려면연결 끊기를누릅니다.

    오디언스가 활성화된 플레이어 만들기

    비디오 참여 데이터는 시청자 지원 비디오 플레이어를 사용하는 경우에만 Marketo로 전송됩니다. 오디언스가 활성화된 브라이트코브 플레이어 생성에 대한 자세한 내용은오디언스가 활성화된 플레이어 만들기를참조하십시오.

    시청자 식별

    Marketo 통합이 있는 오디언스가 활성화된 플레이어는 사용자의 브라우저에서 생성된 Munchkin 쿠키 (_mkto_trk) 를 참조합니다. 이 값은 Audience 모듈의 보기 데이터에 캡처되며 보기 데이터를 Marketo에 동기화하는 데 사용됩니다.

    사용자의 이메일 주소는 비디오를 표시하는 페이지의bcemail URL 매개 변수로 전달할 수 있습니다. 자세한 내용과 예제는대상 모듈에서 뷰어를 식별하는 방법을참조하십시오.

    통합 확인

    통합이 구성되면 통합을 테스트하여 비디오 보기 데이터가 Marketo로 전송되는지 확인합니다. 통합 확인에 대한 자세한 내용은Marketo REST 통합 테스트 가이드를참조하십시오.
