    비디오 클라우드와 Salesforce의 통합

    이 항목에서는 Salesforce에서 사용할 대상 모듈을 구성하는 방법에 대해 설명합니다.
     

    Audience 모듈을 사용하면 비디오 클라우드가 브라이트코브 플레이어의 시청 데이터를 Salesforce로 동기화할 수 있습니다. Salesforce에서 이 데이터는 캠페인 로직, 보고서 및 개인 설정 커뮤니케이션에 사용할 수 있습니다. Salesforce와의 Audience 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

    • 비디오 조회수 데이터 추적을 지원하는 브라이트코브 플레이어 플러그인
    • 비디오 클라우드에서 Salesforce로 추적 데이터를 전송하는 동기화 프로세스

    요구 사항

    Salesforce에서 대상 모듈을 사용하려면 다음을 수행해야 합니다.

    • 프로페셔널 또는 엔터프라이즈 비디오 클라우드 계정
    • Salesforce 계정이 있는 경우 Salesforce 샌드박스 계정은 지원되지 않습니다.
    • Audience 모듈을 열 때 나타나는 초기 시작 화면에서 필요한 정보를 제출하여 계정에 대한 Audience 모듈을 활성화하십시오.
    • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
    • API 사용권한이 있는 Salesforce.com 계정이 있고 보기 데이터를 전송하도록 선택한 Salesforce 객체를 읽고 쓸 수 있는 역할에 속해 있습니다.
    • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내부에서 실행되므로 상위 페이지 URL 매개 변수에 액세스 할 수 없으므로 표준 임베드 (iframe) 게시 코드가 작동하지 않습니다.
    • Pardot 통합을 사용하려면 다음에 설명 된 단계를 완료해야합니다.   Pardot 통합을위한 사용자 지정 Salesforce 개체 만들기 이야기.

    특징

    비디오 클라우드 - Salesforce 통합의 주요 기능은 다음과 같습니다.

    • 데이터 형식 - 비디오 클라우드 보기 데이터가 Salesforce 객체에 매핑됨
    • 데이터 전송 - 비디오 클라우드에서 매시간 Salesforce로 데이터 보기 전송
    • 잠재 고객 양식 - 시청자가 동영상을 시청할 때 자동으로 표시되는 Salesforce 양식을 만듭니다. 자세한 내용은Salesforce용 사용자 정의 가망 고객 양식 생성을참조하십시오.

    연결 설정

    다음 단계에 따라 비디오 클라우드와 Salesforce 간의 연결을 설정합니다.

    1. 대상 모듈을 열고 왼쪽 탐색에서데이터 연결을 클릭합니다.
    2. 새 연결 추가를클릭합니다.
    3. 통합으로Salesforce를선택합니다.
      참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
    4. (선택 사항) 연결 이름을입력합니다.
    5. 연결을클릭합니다.
    6. Salesforce에 연결을클릭합니다.
    7. 로그인 화면이 나타나면 Salesforce 사용자 이름및암호를입력합니다. 로그인을클릭합니다.
    8. 로그인에 성공하면Salesforce용 브라이트코브 비디오 연결앱을 설치하라는 메시지가 표시됩니다. 허용을클릭합니다.

    이제 Salesforce에 대한 비디오 클라우드 연결이 설정되어 사용할 준비가 되었습니다.

    통합 구성

    비디오 클라우드와 Salesforce로의 연결이 완료되면 통합을 구성할 수 있습니다. 통합을 설정할 때 사용된 계정에 대한정보를 보려면 Salesforce 사용자정보를 클릭합니다.

    1. 뷰어 ID를 구성하려면 뷰어를 식별하는방법 구성링크를 클릭한 다음 다음 옵션을 선택합니다.
      • Pardot - Pardot는 시청자를 식별하는 데 사용됩니다. 아래의Pardot 통합 구성섹션을 참조하십시오.
      • 설명서 -당신의 개발 직원은 로직을 추가 할 책임이 있습니다.  시청자 식별 .
    2. 매핑 유형을선택하고데이터 매핑을 구성합니다 .
    3. 기본적으로연결 상태는사용 가능으로 설정됩니다 . 연결 상태에따라 Video Cloud 비디오 참여 데이터가 Salesforce로 전송되는지 여부가 결정됩니다.
      • 사용 - 데이터가 수집되어 Salesforce로 전송됩니다.
      • 사용 안 함 - 데이터가 수집되지 않습니다 (통합이 비활성화됨).
    4. 기본적으로,  알 수없는 시청자에 대한 데이터 추적 될거야 장애인 . 이 설정은 뷰어를 알 수 없는 경우 비디오 참여 데이터를 Video Cloud에 저장할지 여부를 결정합니다. 알려진 뷰어에 대한 데이터만 Salesforce로 전송됩니다.
      • 사용  - 모든 시청자의 데이터가 수집되어 비디오 클라우드에 저장됩니다.
      • 사용 안 함  - 알 수 없는 시청자의 데이터가 비디오 클라우드에 저장되지 않음
    5. 데이터 보존 정책을 선택합니다. 데이터 보존 정책에 따라 시청 데이터와 잠재 고객 양식 데이터 (Audience 리드 양식을 사용하는 경우) 가 Video Cloud 계정에 보존되는 기간이 결정됩니다. Video Cloud에 데이터를 보관하면 중단된 동기화로부터 복구할 수 있습니다. 또한 Audience 는 이벤트가 데이터 보존 기간이 경과할 때까지 동기화에 실패한 이벤트를 계속 다시 시도합니다 (일반적으로 시청자가 동영상을 볼 때 알려진 연락처가 아니기 때문에). 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다. Video Cloud에 사용자 데이터를 보관할 필요는 없습니다. 데이터는 데이터 보존 기간이 지나면 삭제됩니다.

    파라도트 통합 구성하기

    브라이트코브 오디언스가 세일즈포스에 연결되면 Pardot를 사용하여 시청자를 식별하도록 선택할 수 있습니다. 이 섹션에서는 Pardot를 사용하여 뷰어를 식별하는 단계에 대해 설명합니다.

    참고: Pardot 통합을 구성하기 전에Pardot 통합을 위한 사용자 정의 Salesforce 객체 만들기항목에 설명된 단계를 완료해야 합니다.
    참고: 브라이트코브는 브라이트코브 캠페인과의 통합을 구성할 때 사용할 전용 Pardot 사용자 계정을 만들 것을 권장합니다.
    1. 뷰어를 식별하는 방법 구성링크를 클릭합니다.
    2. Pardot에 로그인하는 데 사용하는이메일과비밀번호를입력합니다.

    파라도 API 키 가져오기

    API 키 필드의 값을 가져오려면 다음과 같이 하십시오.

    1. 파라도트에 로그인합니다.
    2. 톱니바퀴 아이콘 ( ) 을 클릭한 다음설정을클릭합니다.
    3. 페이지 상단에 있는내 프로필링크를 클릭합니다.
    4. 값을 API 사용자 키필드에 붙여 넣습니다. 사용자 키들.

    추적 ID 가져오기

    에 대한 가치를 얻으려면 piAld 추적과 piCld 추적필드에서 다음 단계를 따르십시오.

    1. 왼쪽 탐색에서마케팅아이콘을 클릭한 다음캠페인을클릭합니다.
    2. 캠페인 이름을 클릭합니다 (또는 캠페인이 없는 경우 캠페인 추가).
      참고: 어떤 캠페인을 선택하든 브라이트코브 캠페인은 뷰어를 식별하기 위해 쿠키를 얻기 위해 두 값이 모두 필요합니다.
    3. 추적 코드 조회를클릭합니다. PiAid및PiCID값이 표시됩니다.
    4. 값을 복사하여 각 필드에 붙여 넣습니다.
      참고: 브라이트코브 오디언스는 PiAid 및 PiCID 필드를 검증하지 않지만 잘못된 값을 입력하면 브라이트코브 오디언스가 뷰어를 식별할 수 없습니다.
    5. 파라도트에 연결 을클릭합니다.

    일반적으로 Pardot 통합 유형을 사용하는 경우고급매핑 유형을 선택한 다음 뷰 필드를 사용자가 만든Salesforce 사용자 정의 개체에매핑합니다.

    데이터 매핑 구성

    데이터 매핑은 비디오 데이터가 Salesforce 객체의 필드에 매핑되는 방식을 결정합니다. 기본적으로 통합은 Salesforce의Task객체에 일부 비디오 데이터를 매핑하는단순매핑을 사용하여 구성됩니다.

    다음 단계에 따라 데이터 매핑을 사용자 정의합니다.

    1. Salesforce 개체의 필드에 데이터가 매핑되는 방법 구성을클릭하거나매핑링크를 클릭합니다.
    2. 매핑 유형에대해고급을선택합니다.
    3. Salesforce 개체드롭다운 목록에서 비디오 데이터를 매핑할 개체를 선택합니다.
    4. 적절한보기 필드를 Salesforce 필드에 매핑합니다.
    5. 저장을 클릭하여 변경 사항을 저장합니다.

    Salesforce와의 통합을 제거하려면연결 끊기를클릭합니다. Audience 모듈은 마케팅 플랫폼에 대한 여러 연결을 지원합니다. 추가 통합을 구성하려면 새 연결 추가를클릭합니다.

    오디언스가 활성화된 플레이어 만들기

    비디오 참여 데이터는 시청자 지원 비디오 플레이어를 사용하는 경우에만 Salesforce로 전송됩니다. 오디언스가 활성화된 브라이트코브 플레이어 생성에 대한 자세한 내용은오디언스가 활성화된 플레이어 만들기를참조하십시오.

    추적 ID 전달

    대상 모듈은 추적 ID 값을 Salesforce에 전달하는 기능을 지원합니다. 추적 ID는 캠페인 ID를 추적하는 데 사용할 수 있습니다. 추적 ID는 오디언스가 활성화된 플레이어를 표시하는 URL에 추가하거나 플레이어 임베드 코드에 추가할 수 있습니다.

    • 예: URL에 추가됨
      http://www.myurl.com/video.html?bctrackingid=12345
    • 예: 플레이어 임베드 코드에 추가됨
      <비디오 데이터 - 비디오 ID="6033402539001" 
         데이터 계정 = “1486906377” 
         데이터 플레이어 = “기본값” 
         데이터 포함 = “기본값” 
         데이터 애플리케이션 ID 
         클래스="비디오-js”
         데이터 BC 추적 ID="12345" 
         > <컨트롤/비디오 >
  <스크립트 스크립트 스크립트: “> </” >

    그러면 대상 그룹이 동기화되는 다른 필드로 Salesforce에 값을 전달합니다.

    시청자 식별

    Salesforce 연결이 수동 뷰어 식별을 사용하도록 구성된 경우 뷰어를 식별하는 것은 사용자에게 달려 있습니다.  뷰어 추적이 이루어지려면 사용자 ID가 비디오를 표시하는 페이지에서 URL 매개 변수로 전달되어야 합니다. 두 개의 URL 매개 변수가 지원됩니다.

    • bcemail - 최종 사용자의 이메일 주소를 전달하려면 (예를 들어, 이 항목참조)
    • bcsfid - 시청자의 세일즈포스 ID를 전달하려면

    이 시나리오에서 비디오 랜딩 페이지는 추적을 위해 고유하게 생성된 URL로 생성됩니다. Audience 모듈은 URL 매개 변수를 사용하여 사용자 서버 측을 조회하고 보기 데이터를 동기화합니다.

    플레이어/플러그인이 URL 매개 변수를 읽으려면 고급 임베드 (페이지 내) 게시 코드를 사용해야 합니다. 플러그인이 iframe 내부에서 실행되므로 매개 변수가있는 상위 페이지 URL에 액세스 할 수 없으므로 표준 포함 (iframe) 게시 코드가 작동하지 않습니다.

    참고: 기본적으로 연락처 및 리드 테이블은 뷰어의 Salesforce ID를 찾는 데 사용됩니다. Salesforce에서 계정 테이블을 대신 사용하도록 대상을 구성할 수 있습니다. 이를 활성화하려면브라이트코브 지원에 문의하여요청하십시오. 예를 들면 다음과 같습니다. “뷰어 ID를 찾을 때 Salesforce 계정 테이블을 사용하도록 내 대상 계정을 구성하십시오.”

    세일즈포스에서 비디오 데이터 보기

    시청자가 활성화된 플레이어에서 비디오를 볼 때 비디오 보기 데이터가 Salesforce에 동기화됩니다. 단순데이터 매핑 옵션을 선택한 경우 비디오 보기 데이터가 Salesforce 내에서작업항목으로 나타납니다. 고급데이터 매핑 옵션을 사용한 경우 매핑 구성 시 선택한 Salesforce 개체의 일부로 데이터 보기가 나타납니다. 다음 예에서는고급데이터 매핑이 사용되고 비디오 필드가 사용자 정의BC 비디오 보기 Salesforce 객체에 매핑되었습니다.
