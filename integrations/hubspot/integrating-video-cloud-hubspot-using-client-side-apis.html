페이지 내용

    클라이언트측 API를 사용하여 비디오 클라우드와 HubSpot의 통합

    이 항목에서는 클라이언트측 API를 사용하여 HubSpot에서 사용하도록 대상 모듈을 구성하는 방법에 대해 설명합니다.

    Audience 모듈을 사용하면 비디오 클라우드가 브라이트코브 플레이어의 시청 데이터를 HubSpot으로 동기화할 수 있습니다. HubSpot에서 이 데이터는 세분화, 캠페인 로직, 보고서, 잠재 고객 점수 및 개인 설정 커뮤니케이션에 사용할 수 있습니다.

    비디오 클라우드를 HubSpot과 통합하기 위한 두 가지 옵션이 있습니다.

    • 표준 (REST API) 메서드 사용 - 이 메서드를 사용하면 대상 사용자가 알려진 뷰어에 대한 보기 데이터만 추적합니다. 기본적으로 리드 양식은 알 수 없는 시청자에게만 표시됩니다. 이러한 유형의통합에 대한 자세한 내용은 REST API를 사용하여 비디오 클라우드와 HubSpot의통합을 참조하십시오.
    • 높은 볼륨 (클라이언트 측 API) 메서드 사용 - 이 메서드를사용하면 Audient가 모든 뷰어에 대한 보기 데이터를 추적합니다. 이 방법에 대해서는 이 항목에서 다룹니다.

    HubSpot과의 Audience 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

    • 비디오 조회수 데이터 추적을 지원하는 브라이트코브 플레이어 플러그인
    • 비디오 클라우드에서 HubSpot으로 추적 데이터를 전송하는 동기화 프로세스

    요구 사항

    HubSpot에서 대상 모듈을 사용하려면 다음을 수행해야 합니다.

    • 허브스팟 계정이 있는 경우
    • 프로페셔널 또는 엔터프라이즈 비디오 클라우드 계정
    • Audience 모듈을 열 때 나타나는 초기 시작 화면에서 필요한 정보를 제출하여 계정에 대한 Audience 모듈을 활성화하십시오.
    • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
    • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내부에서 실행되므로 표준 임베드 (iframe) 게시 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수 없으며 올바른 쿠키를 얻지 못할 수 있습니다.

    특징

    비디오 클라우드 - HubSpot 통합의 주요 기능은 다음과 같습니다.

    • 잠재 고객 양식 - 시청자가 동영상을 볼 때 자동으로 표시되는 HubSpot 양식을 만듭니다. 자세한 내용은HubSpot에 대한 사용자 정의 잠재 고객 양식 생성을참조하십시오.
    • 데이터 형식 - 데이터 보기는 사용자 정의타임라인 이벤트 유형으로제공됩니다. HubSpot에 복제되는 데이터는 다음과 같습니다.
      • 비디오 이름
      • 비디오 ID
      • 계정 ID (비디오 클라우드)
      • 페이지 URL (참조 페이지의 URL)
      • 플레이어 ID (비디오 클라우드 플레이어)
      • 감시률 (25%, 50%, 75%, 95%)
    • 데이터 전송 - 데이터를 보는 것은 자동화된 프로세스를 사용하여 HubSpot으로 전송됩니다. 그러나 HubSpot 플랫폼의 일부 보고서는 새로 고치는 데 약간의 시간이 걸릴 수 있습니다.
    • 데이터 보존 - 수집된 데이터를 최대 6개월 동안 Video Cloud에 선택적으로 보존할 수 있음

    연결 설정

    비디오 클라우드와 HubSpot 간의 연결을 설정하려면 다음 단계를 따르십시오.

    비디오 클라우드와 HubSpot 간의 연결 설정

    1. 대상 모듈을 열고 왼쪽 탐색에서데이터 연결을 클릭합니다.
    2. 새 연결 추가를클릭합니다.
    3. 통합으로HubSpot을선택합니다.
      참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
    4. (선택 사항) 연결 이름을입력합니다.
    5. 연결을클릭합니다.
    6. 대용량 (클라이언트 쪽 API)을 선택합니다.
    7. 저장을 클릭합니다.

    이제 HubSpot에 대한 비디오 클라우드 연결이 설정되었습니다.

    통합 구성

    비디오 클라우드와 HubSpot 연결이 이루어지면 다음 단계에 따라 통합을 구성합니다.

    1. 통합하려면 HubSpot 계정 ID가 필요합니다. 빈링크를 클릭하고 HubSpot 계정 ID를 입력한 다음저장을클릭합니다.
      참고: 동일한 계정 ID를 가진 여러 HubSpot 연결을 사용하는 것은 지원되지 않습니다.
    2. 기본적으로연결 상태는사용 가능으로 설정됩니다 . 연결 상태에따라 비디오 클라우드 비디오 참여 데이터가 HubSpot으로 전송되는지 여부가 결정됩니다.
      • 사용 - 데이터가 수집되어 HubSpot으로 전송됩니다.
      • 사용 안 함 - 데이터가 수집되지 않습니다 (통합이 비활성화됨).

    HubSpot과의 통합을 제거하려면연결 끊기를클릭합니다. Audience 모듈은 마케팅 플랫폼에 대한 여러 연결을 지원합니다. 추가 통합을 구성하려면 새 연결 추가를클릭합니다.

    오디언스가 활성화된 플레이어 만들기

    비디오 참여 데이터는 시청자 지원 비디오 플레이어를 사용하는 경우에만 HubSpot으로 전송됩니다. 오디언스가 활성화된 브라이트코브 플레이어 생성에 대한 자세한 내용은오디언스가 활성화된 플레이어 만들기를참조하십시오.

    추적 ID 전달

    대상 모듈은 추적 ID 값을 HubSpot에 전달하는 기능을 지원합니다. 추적 ID는 캠페인 ID를 추적하는 데 사용할 수 있습니다. 추적 ID는 오디언스가 활성화된 플레이어를 표시하는 URL에 추가하거나 플레이어 임베드 코드에 추가할 수 있습니다.

    • 예: URL에 추가됨
      http://www.myurl.com/video.html?bctrackingid=12345
    • 예: 플레이어 임베드 코드에 추가됨
      <비디오 데이터 - 비디오 ID="6033402539001" 
         데이터 계정 = “1486906377” 
         데이터 플레이어 = “기본값” 
         데이터 포함 = “기본값” 
         데이터 애플리케이션 ID 
         클래스="비디오-js”
         데이터 BC 추적 ID="12345" 
         > <컨트롤/비디오 >
  <스크립트 스크립트 스크립트: “> </” >

    그러면 청중은 값을 동기화된 다른 필드로 HubSpot에 전달합니다.

    시청자 식별

    HubSpot 통합의 경우 플러그인은 사용자의 브라우저에hubspotutk 쿠키를 삭제하는 HubSpot의자바 스크립트 추적 코드를 주입합니다. 이 쿠키는 HubSpot 내에서 방문자의 신원을 추적하고 브라이트코브 시청 데이터를 연락처와 연결하는 데 사용됩니다. 플레이어가로드 될 때 사용자가 이미hubspotutk쿠키를 가지고있는 경우, 청중은 HubSpot에 API 요청을 만들고 쿠키 값이 알려진 연락처에 해당하는지 묻습니다. 그렇다면 오디언스는 뷰 데이터를 해당 사용자와 연결합니다.
