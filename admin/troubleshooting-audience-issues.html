페이지 내용

    고객 문제 해결

    이 항목에서는 갤러리 환경을 통해 수집된 잠재 고객 양식 데이터를 내보내는 방법에 대해 알아봅니다.

    자세한 정보 로깅 사용

    자세한 로깅을 활성화하면 브라우저 콘솔에서 Audience 관련 정보를 볼 수 있습니다. 이 기능은 잠재고객 문제를 디버깅할 때 유용합니다. 자세한 로깅은 오디언스 지원 플레이어로 페이지의 bcverbose=1 URL에 추가하거나 다음"verbose": true옵션으로 Audience 플러그인을 구성하여 활성화할 수 있습니다.

    로그 메시지는 콘솔에서 볼 수 있습니다.

    리드 양식 문제

    다음은 리드 양식에서 본 몇 가지 일반적인 문제와 질문입니다.

    양식이 제출된 후에도 Marketo Munchkin 통합에 대해 리드 양식이 계속 표시되는 이유는 무엇입니까?

    Munchkin 통합에는 사용자가 알고 있는지 여부를 묻는 데 사용할 수있는 API가 없습니다. 알려진 사용자인지 알 수 없으므로 항상 리드 양식을 표시합니다. Marketo REST 통합은 알려진 사용자와 알 수 없는 사용자를 확인할 수 있도록 API를 제공합니다. Marketo REST 통합으로 전환하는 것이 권장되며 이 문제를 방지할 수 있습니다.

    내 사용자 지정 리드 양식이 제출시 리디렉션되도록 설정되어 있습니다. 양식을 제출한 후 1~2초 동안 리디렉션되는 페이지가 표시되는 이유는 무엇입니까?

    리드 양식을 해제하기 전에 리디렉션 된 페이지가 완전히로드 될 때까지 기다려야합니다. 한 가지 해결책은 가벼운 페이지 (이미지 또는 JavaScript가 많지 않음) 또는이 문제를 해결하기 위해 특별히 설정 한 빈 페이지 (위치https://map.brightcove.com/lead/complete ) 로 리디렉션하는 것입니다.

    내 리드 양식을 확인 HTML로 구성했습니다. 제출 후 몇 초 동안 검은 화면이 표시되는 이유는 무엇입니까?

    HTML의 텍스트 색상을 변경해야합니다. 확인 HTML이 표시 될 수 있습니다. 기본 텍스트 색상이 검정색이기 때문에 볼 수 없습니다.

    제출 후 HubSpot 리드 양식이 취소되지 않는 이유는 무엇입니까?

    HubSpot의 리드 양식이 로 리디렉션되도록 구성되어 있는지 확인합니다http://map.brightcove.com/lead/complete . 확인 메시지를 표시하려면 HubSpot이 아닌 Audience 내에서 이 메시지를 구성합니다.

    리드 양식이 표시되지 않는 이유는 무엇입니까?

    가망 고객 양식이 표시되지 않는 경우 먼저 가망 고객 양식을 표시하기 위한 모든 기준이 충족되었는지 확인합니다. 시청하는 동영상에 따라 달라지는 기준은 페이지 새로 고침에서 작동하지 않습니다. 다음 동영상이 같은 플레이어에서 자동으로 대기열에 배치되는 사용 사례를 위한 것입니다.

    1. Audience 플러그인이 플레이어에 추가되었는지 확인합니다. 자세한 내용은오디언스가 활성화된 플레이어 만들기를 참조하십시오.
    2. 통합이 올바르게 설정되었고 플러그인으로 전송되었는지 확인합니다.
    3. 리드 양식이 해당 플레이어에게 추가되었는지 확인합니다. 플러그인이 설정에 대해 만드는 호출에 대한 응답에는 리드 양식이 플레이어에 추가 된 경우 LeadForm 속성 아래에 리드 양식 표시 정보가 포함되어야합니다.
    4. 뷰어가 아직 리드가 아닌지 확인합니다. 잠재 고객 양식은 시청자가 마케팅 자동화 플랫폼 (MAP) 의 잠재 고객이 아니거나 아직 잠재 고객 양식을 작성하지 않은 경우에만 표시됩니다. 플러그인은 시청자가 리드인지 확인하기 위해 Audience 서버에 대한 별도의 JSONP 호출을 만듭니다. 뷰어가 리드 인 경우 서버는 상태 1 및 뷰 데이터를 MAP에 동기화하는 데 유용 할 수있는 다른 식별자로 응답합니다. 뷰어가 리드가 아닌 경우 서버는 상태 0으로 응답합니다.
      • 리드가 아닌 뷰어에 대한 응답 예: 
        /**/ JSON 콜백_2935650618 == '함수'&&
 JSON 콜백_2935650618 (상태: 0});
      • 잠재 고객인 시청자에 대한 응답 예: 
        /**/ JSON 콜백의 유형_45936049288 === '함수 '&&
 JSON콜백_45936049288 ({"상태”: “엘로콰어”: “5", “내부ID”: “CTPB0000000005"});

    동기화 문제

    다음은 우리가 본 몇 가지 일반적인 동기화 문제입니다.

    Marketo REST 보기 이벤트가 활동 유형 ID '0000'을 (를) 찾을 수 없다는 오류와 동기화되지 않습니다.

    이 문제는 통합과 연결된 사용자 지정 Marketo 활동이 삭제되었을 때 발생합니다. Marketo REST 통합이 생성되면 활동을 가져오고, ID를 저장하고, 해당 ID를 사용하여 이벤트를 동기화합니다. 새로운 브라이트코브 Video View 활동이 생성되었다고 가정할 때 해결 방법은 통합 연결을 끊었다가 다시 만들어 새 활동의 ID를 선택하는 것입니다.

    동기화되지 않는 이벤트 보기

    MAP로 통합이 올바르게 설정되면 Audience-Enabled 플레이어에서 데이터를 보는 것이모든 최근 활동페이지에 채워지고 각 MAP의 알려진 리드에 해당하는 이벤트가 결국 도구와 동기화되어야 합니다.

    먼저모든 최근 작업페이지에 뷰 데이터가 표시되는지 확인합니다.

    뷰 이벤트가 나열되지만 MAP에 없는 경우 API를 통해 MAP로 뷰 데이터를 가져올 수 있는 권한이 없는 사용자로 Audience 권한을 부여했거나 통합이 올바르게 구성되지 않았을 수 있습니다. Audience 홈페이지에는 최근 보기 이벤트 동기화 작업의 상태가 나열됩니다. 뷰 이벤트가 MAP에 동기화되지 않는 이유를 나타내는 피드백을 사용할 수 있어야 합니다.

    보기 이벤트가 오디언스에 표시되는 경우상태열이동기화되었는지확인합니다. 이는 오디언스가 데이터를 MAP에 성공적으로 전달할 수 있음을 나타냅니다. 알 수없는 연락처 상태가 표시되면 사용자를 알 수 없고 MAP에서해당 사용자를 받을 수 없음을 의미합니다. 대부분의 지도는 알려진 사용자에 대한 타사 데이터만 허용합니다. 해결책은 MAP에서 생성 된 사용자 지정 리드 양식을 사용하여 실제로 알려진 사용자인지 확인하는 것입니다.

    보기 이벤트가 목록에 없는 경우 오디언스가 활성화된 플레이어로 설정되지 않았을 수 있습니다. 개발자 도구가 활성화된 브라우저를 사용하여 문제가 있는 비디오가 포함된 페이지를 방문하여 Audience 플러그인에서 특정 HTTP 호출이 이루어지는지 확인합니다.

    플레이어가로드 될 때 플러그인은 Audience 구성 끝점을 호출하여 통합을 기반으로 플러그인의 동작을 결정하는 매개 변수를 얻어야합니다.

    네트워크 트래픽에서 이 호출이 누락되면 플레이어에 통합이 할당되지 않았음을 의미합니다. 이 작업을 수행하는 방법에 대한 자세한 내용은대상 사용 플레이어 만들기를참조하십시오.

    이 호출은 JSONP를 사용하여 수행됩니다. JSONP 응답이 빈 객체인 경우 embed 코드가 잘못되어 잘못된 계정 ID를 지정할 수 있습니다. 구성 요청에 계정 및 플레이어 URL 매개 변수가 모두 포함되지 않거나 Audience-특정 데이터베이스에서 지정된 계정을 검색할 수 없는 경우 (URL/embed 코드에 잘못된 계정이 지정되었을 가능성이 높으며데이터베이스).

    /**/ JSON콜백의 유형_78754666587 == '함수' && JSON 콜백_78754666587 ({});

    개체에 ID 매개 변수가 있지만 통합 매개 변수가 없는 경우 사용자가 마케팅 자동화 도구와의 통합을 설정하지 않았으며 보기 이벤트가 기록되지 않습니다.

    오디언스가 활성화된 플레이어를 게시할 때 표준 임베드 (iframe) 코드가 사용된 경우에도 데이터 동기화 문제가 발생합니다. 플레이어를 게시할 때 고급 포함 (페이지 내) 코드를 사용합니다. Audience 플러그인이 iframe 내부에서 실행되므로 상위 페이지 쿠키 또는  URL 매개 변수에 액세스 할 수 없으므로 표준 임베드 (iframe) 게시 코드가 작동하지 않습니다.

    연결 문제

    Audience 모듈은 Eloqua 통합을 설정할 수 없다고 말합니다.

    Eloqua에 대한 요구 사항을확인하십시오. IP 화이트리스트 보안 기능을 사용하는 경우 Audience - Eloqua 트래픽을 허용하기 위해 지정된 IP 주소를 화이트리스트하지 않는 한 Audience Eloqua와 통신할 수 없습니다. 자세한 내용은브라이트코브 지원에문의하십시오.

    잠재 고객이 Marketo REST 사용자 지정 활동에 필드가 누락되었다고 말하지만 Marketo에서 보면 모든 필드가 있음을 알 수 있습니다.

    확인해야 할 몇 가지 사항이 있습니다.

    1. 필드 이름은 정확히 일치해야 합니다. “이벤트 1"은 “이벤트 1"과 동일하지 않습니다.
    2. 활동은 초안 상태가 아니라 승인되어야 합니다.
    3. 사용자 지정 활동을 삭제하고 새 Marketo REST 통합을 설정하지 않고 다시 만든 경우 이전 통합의 연결을 끊고 새 통합을 만들어야 합니다. 사용자 지정 활동을 조회하고 각 통합에 대한 ID 값을 저장합니다. ID를 삭제하여 변경하는 경우 새 ID를 자동으로 선택할 수 없습니다.
    4. 최근에 활동에 새 필드가 추가된 경우 Marketo 시스템에 변경 사항이 적용되기까지 최대 24시간이 걸릴 수 있습니다.
