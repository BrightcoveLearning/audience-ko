    사용자 지정 지표 및 이벤트 사용

    이 항목에서는 이벤트를 보기 위해 사용자 지정 이벤트와 지표를 연결하는 방법을 배웁니다.

    Audience Player 플러그인에는 사용자 지정 이벤트와 지표를 연결하여 이벤트를 볼 수 있는 클라이언트 측 API가 포함되어 있습니다. 다음 예에서는 사용자 지정 이벤트 및 지표를 추적하는 방법을 간략하게 설명합니다.

    1. 브라이트코브 플레이어에 대한 참조를 얻으십시오. 이 예제에서는id의myPlayerID가 플레이어 embed 코드에 추가되었다고 가정합니다. 
        
var player = videojs(‘myPlayerID’);
    2. 오디언스 준비 이벤트를 들어보세요audience:ready . 
        
player.on(‘audience:ready’, function() {
  // player.audience is now available for use
});
    3. 문자열 값을 추적하려면 를player.audience.track(‘event[1-3]’, ‘value’)호출합니다. 숫자 값을 추적하려면 를player.audience.track(‘metric[1-3]’, value)호출합니다. 반환 값은 이벤트가 성공적으로 전송되었는지 여부를 나타내는 부울입니다. 
        
// Store the string value “foo” in the slot event1
player.audience.track(‘event1’, ‘foo’);

// Store the string value “bar” in the slot event2
player.audience.track(‘event2’, ‘bar’);

// Store the decimal value 15.08 in slot metric1
player.audience.track(‘metric1’, 15.08);

// Replace the value in slot metric1 with the number 10,
// and validate that the call was sent successfully
var result = player.audience.track(‘metric1’, 10);

if (result) {
   console.log(‘metric sent successfully’);
}

    마케팅 자동화 플랫폼과의 동기화

    뷰 이벤트에 저장된 사용자 지정 이벤트/지표는 필드가 사용자 지정 활동 또는 CDO에 정의되어 있는 경우 Marketo REST 및 Eloqua에 대해 동기화됩니다. 또한 대상 그룹에서는 사용자 지정 이벤트/지표를 HubSpot에 동기화하고 통합에서고급매핑 유형을사용하고 이벤트/지표가 무언가에 매핑되는 경우 Salesforce에 동기화합니다.

    마르케토

    Marketo REST 통합의 경우 게시자는 Marketo의브라이트코브 비디오 뷰사용자 지정 활동에 사용자 지정 이벤트 및 지표 필드가 있는지 확인해야 합니다.

    필드가 브라이트코브 비디오 보기 개체에 없는 경우 다음을 수행합니다.

    1. 브라이트코브 비디오 보기사용자 지정 활동을 업데이트하여 필드를 추가하거나
    2. Marketo 지원에 문의하여 필드를 추가하도록 요청합니다.
    3. 필드가 추가된 후 오디언스로 돌아가서업데이트 완료버튼을 누르면 오디언스가 새로 추가된 필드에 액세스할 수 있습니다.

    Oracle Eloqua

    Eloqua 통합의 경우 게시자는BrightCoveVideoView사용자 지정 개체에 사용자 지정 이벤트 및 메트릭 필드가 있는지 확인해야 합니다.

    필드가 없으면 다음 중 하나를 사용하여 필드를BrightCoveVideoView객체에 추가해야 합니다.

    1. Eloqua 계정을 다시 연결하거나
    2. 데이터 매핑 업그레이드 단추 사용

    영업

    Salesforce의 경우 대상 연결에서고급매핑 유형을사용해야 합니다. 이벤트및메트릭필드는 Salesforce 개체에 매핑할 수 있습니다.

    참고

    • 이벤트 슬롯 값의 문자열 길이 제한은 250자입니다.
    • 단일 보기 세션 중에 동일한 이벤트 또는 메트릭 슬롯에 여러 값이 전송되면 슬롯의 기존 값이 새 값으로 대체됩니다. 다음 예에서 event1에 저장된 최종 값은 “click3”입니다. 
        
player.audience.track(‘event1’, ‘click1’);
player.audience.track(‘event1’, ‘click2’);
player.audience.track(‘event1’, ‘click3’);
    • 세션에 대한 뷰 이벤트가 아직 존재하지 않으면 하나가 생성됩니다.
    • 의 결과가track() false이면 이벤트를 보내지 못했음을 나타내는 경우 Audience 플러그인에서 자세한정보 표시 옵션을 활성화하여콘솔 로깅을 확인할 수 있습니다.

    이벤트 및 지표 보기

    사용자 지정 이벤트 및 지표가 대상 UI에 나타납니다. 왼쪽 탐색에서모든 최근 활동을클릭합니다. 사용자 정의 이벤트 또는 지표가 기록된 경우상호 작용 이벤트 표시링크가 활성화됩니다. 링크를 눌러 이벤트/메트릭 데이터를 표시합니다.

    자세한 정보 로깅 사용

    자세한 로깅은 플레이어와 함께 페이지의 bcverbose=1 URL에 추가하거나"verbose": true옵션으로 Audience 플러그인을 구성하여 활성화할 수 있습니다.
