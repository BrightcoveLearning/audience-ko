    대상 모듈이 뷰어를 식별하는 방법

    이 항목에서는 Audience 모듈이 시청자를 식별하려고 시도하는 여러 가지 방법에 대해 학습합니다.

    Audience 모듈은 다양한 데이터 소스를 사용하여 뷰어를 식별하려고 시도합니다.

    • 청중 쿠키
    • 오라클 엘로쿠아 GUID
    • 마르케토 먼치킨 쿠키
    • 세일즈포스 ID URL 매개 변수
    • 세일즈포스 ID HTML 속성
    • 허브 스팟 UTK 쿠키
    • 전자 메일 매개 변수

    Audience 모듈은추적 금지관련 사용자의 브라우저 설정을 준수합니다.

    참고: 사용자 지정 통합을 사용하는 경우 뷰어를 식별할 책임이 있습니다. 자세한 내용은대상 모듈과의 사용자 정의 통합 작성을참조하십시오.

    청중 쿠키

    시청자가 활성화된 플레이어는 자신의 계정에 대해 오디언스 트래킹이 활성화되어 있고 플레이어가 이를 사용하는 경우 제3자 쿠키를 설정하고 이를 사용하여 시청 데이터를 동일한 사용자와 연결합니다. 사용자가 오디언스 리드 양식을 작성하는 경우 오디언스는 해당 뷰어에 대한 프로필을 만들고 모든 시청 데이터는 해당 프로필과 연결됩니다.

    관객이 설정할 쿠키의 이름은_bc_uuid입니다 .

    오라클 엘로쿠아 GUID

    Eloqua 통합이 있는 오디언스가 활성화된 플레이어는 Eloqua API를 호출하여 방문자 ID (주로 Eloqua GUID) 를 얻습니다. 이 ID는 Audience 모듈의 보기 데이터에 캡처되며 보기 데이터를 Eloqua에 동기화하는 데 사용됩니다.

    Eloqua는 자체 쿠키 세트를 사용하며 브라이트코브 플레이어가 Eloqua 통합으로 구성된 경우 Eloqua 쿠키의 GUID를 캡처하고 Eloqua 서버 측과 통신하여 사용자가 누구인지 파악합니다.

    마르케토 먼치킨 쿠키

    Marketo 통합이 있는 오디언스가 활성화된 플레이어는 사용자의 브라우저에서 생성된 Munchkin 쿠키 (_mkto_trk) 를 참조합니다. 이 값은 Audience 모듈의 보기 데이터에 캡처되며 보기 데이터를 Marketo에 동기화하는 데 사용됩니다.

    세일즈포스 ID URL 매개 변수

    Salesforce 통합의 경우 플러그인은bcsfid라는매개 변수의 URL을 확인하여 뷰어를 식별하려고 할 수 있습니다. 이를 통해 Salesforce 사용자는 URL에 잠재 고객의 Salesforce ID를 제공하여 추적을 위해 고유하게 생성된 랜딩 페이지 URL이 포함된 이메일 캠페인을 보낼 수 있습니다.

    세일즈포스 ID HTML 속성

    Salesforce 통합의 경우 플러그인이 다음을 확인하여 뷰어를 식별 할 수없는 경우 bcsfid URL 매개 변수를 검색합니다. data-bc-sfid속성<video> 비디오에 대한 HTML 포함 게시 코드의 요소이며 설정된 경우 해당 값을 리드의 Salesforce ID로 사용합니다.

    허브 스팟 UTK 쿠키

    HubSpot 통합의 경우 플러그인은 사용자의 브라우저에hubspotutk 쿠키를 삭제하는 HubSpot의자바 스크립트 추적 코드를 주입합니다. 이 쿠키는 HubSpot 내에서 방문자의 신원을 추적하고 브라이트코브 시청 데이터를 연락처와 연결하는 데 사용됩니다. 플레이어가로드 될 때 사용자가 이미hubspotutk쿠키를 가지고있는 경우, 청중은 HubSpot에 API 요청을 만들고 쿠키 값이 알려진 연락처에 해당하는지 묻습니다. 그렇다면 오디언스는 뷰 데이터를 해당 사용자와 연결합니다.

    전자 메일 매개 변수

    Audience 모듈은 이메일 주소를 통해 사용자가 누구인지 알 수도 있습니다. 청중은bcemail이라는 URL 매개 변수를 찾습니다. 이는 추적을 위해 고유하게 생성된 랜딩 페이지 URL이 포함된 이메일 캠페인에 유용합니다. 마케팅 플랫폼 또는 MailChimp와 같은 타사 이메일 서비스에서 목록으로 일괄 이메일을 보내는 경우 보내는 플랫폼의 설정에서 이메일 매개 변수를 구성하여 사람들이 클릭하는 링크를 동적으로 업데이트해야 합니다. 그런 다음 오디언스는이 이메일 매개 변수를 사용하여 사용자 서버 측을 조회하고 뷰 데이터를 동기화합니다. 이메일 매개 변수는 플레이어 임베드 코드와 함께 포함될 수도 있습니다.

    URL 예제

    http://go.brightcove.com/be-video-hero-cust?bcemail=msmith@brightcove.com

    Audience 플러그인이bcemail URL 매개 변수를 통해 이메일 주소를 찾지 못하면 HTML 임베드 게시 코드의<video> 요소를 확인합니다. 비디오 요소의데이터 bc-email속성을 검색하고 설정된 경우 이메일 주소로 값을 사용합니다.

    포함 예제

    <비디오

    추적 안 함

     시청자가 활성화된 플레이어는 DNT (추적 금지) 와 관련된 시청자의 브라우저 설정을 준수하며 추적 금지 기능이 활성화되어 있는 경우 일반 쿠키를 사용하거나 설정하지 않습니다. 플레이어 플러그인은 또한 Video Cloud 고객이 모든 시청자 또는 사용자별로 이 플러그인을 끄기를 원하는지 추적하지 않도록 지시할 수 있습니다. 이 작업은 페이지 내장 코드를 사용하고 값을 1로 설정하여 수행됩니다. data-do-not-track

    Oracle Eloqua

    Eloqua 는 요청을 추적하지 않고 (기본적으로 무시) 처리하지 않으며 Eloqua GUID를 반환합니다. 현재 오디언스는 DNT를 볼 때 Eloqua JavaScript API를 사용하지 않는 특별한 방법으로이를 처리하지 않습니다. Eloqua 또한 EU 개인 정보 보호 요구 사항을 충족하는 엄격한 모드를 지원합니다. Eloqua 계정에 대해 이 기능이 활성화되면 양식을 통해 옵트인 (opt in) 하지 않는 한 최종 사용자가 추적되지 않습니다. 두 경우 모두 삭제되는 쿠키는Eloqua 추적 페이지를 탐색할 때 삭제되는 쿠키가 Oracle 문서에 문서화되어 있습니다.

    Marketo

    Marketo는 DNT를 지원하지만 특정 계정에 대해 DNT를 켜거나 끌 수 있도록 고객에게 남겨 둡니다. 자세한 내용은 Marketo 문서“추적 안 함” 브라우저 지원 설정 편집을참조하십시오.

    HubSpot

    HubSpot은이 문서에따라 DNT를 존중합니다.
